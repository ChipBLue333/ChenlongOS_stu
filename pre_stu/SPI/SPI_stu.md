## 1. 协议概览

### 1.1 什么是 SPI？
SPI 是一种全双工、同步、串行、主从结构（Master-Slave）的通信协议。它的硬件实现简单，传输速率通常高于 I2C。

**典型应用：**
- **存储器**：Flash, EEPROM
- **传感器**：加速度计、陀螺仪、ADC/DAC
- **显示器件**：LCD, OLED
- **通信模块**：网卡、蓝牙、无线收发器

### 1.2 核心特点
- **同步通信**：收发双方共用同一根时钟线。
- **全双工**：数据发送和接收可以同时进行。
- **主从架构**：由主机（Master）发起通信并产生时钟。

## 2. 硬件连接与信号线

标准 SPI 通信通常使用 4 根信号线：

| 缩写 | 全称 | 描述 | 方向 |
| :--- | :--- | :--- | :--- |
| **SCLK** | Serial Clock | 主机产生的同步时钟信号 | Master → Slave |
| **MOSI** | Master Out Slave In | 主机发送给从机的数据线 | Master → Slave |
| **MISO** | Master In Slave Out | 从机发送给主机的数据线 | Slave → Master |
| **CS/SS** | Chip/Slave Select | 片选信号（通常低电平有效） | Master → Slave |

### 2.1 拓扑结构

#### 单主单从
```text
  [ Master ]                        [ Slave ]
      SCLK  ---------------------->  SCLK
      MOSI  ---------------------->  SDI
      MISO  <----------------------  SDO
      CS    ---------------------->  CS
```

#### 单主多从 (独立片选)
```text
           +------ SCLK ------+---- Slave 1
[ Master ] +------ MOSI ------+---- Slave 2
           +------ MISO ------+---- Slave 3
           |                  |
           +------ CS1 ------> Slave 1
           +------ CS2 ------> Slave 2
           +------ CS3 ------> Slave 3
```

## 3. 关键时序：CPOL 与 CPHA

这是 SPI 驱动开发中最核心的两个参数，它们共同决定了数据的采样时机。

### 3.1 参数定义
- **CPOL (Clock Polarity)**：时钟极性。定义时钟线在空闲状态下的电平。
- **CPHA (Clock Phase)**：时钟相位。定义在时钟的哪一个边沿进行数据采样。

### 3.2 四种工作模式

| 模式 | CPOL | CPHA | 空闲电平 | 采样时刻 |
| :--- | :--- | :--- | :--- | :--- |
| **Mode 0** | 0 | 0 | 低电平 | 第 1 个边沿（上升沿） |
| **Mode 1** | 0 | 1 | 低电平 | 第 2 个边沿（下降沿） |
| **Mode 2** | 1 | 0 | 高电平 | 第 1 个边沿（下降沿） |
| **Mode 3** | 1 | 1 | 高电平 | 第 2 个边沿（上升沿） |

## 4. 数据交互原理

### 4.1 移位寄存器机制
SPI 的本质是主机和从机之间的**移位寄存器交换**。
1. 主机向移位寄存器写入一个字节。
2. 随着 SCLK 的脉冲，主机的位通过 MOSI 移出，同时从机的位通过 MISO 移入。
3. 8 个时钟周期后，两个寄存器的内容完成互换。

### 4.2 "Dummy Data" (哑数据)
由于位交换是双向的：
- **只读取**：主机必须发送“哑数据”（如 `0xFF` 或 `0x00`）来产生时钟，从机才能将数据移出。

## 5. 软件实现思路 (C 语言示例)

### 5.1 基础发送接收函数
```c
uint8_t spi_transfer(uint8_t data) {
    SPI_DR = data;
    while(!(SPI_SR & RXNE_FLAG));
    return SPI_DR;
}
```

### 5.2 读写寄存器流程
```c
void write_reg(uint8_t addr, uint8_t val) {
    CS_LOW();
    spi_transfer(addr); // 地址
    spi_transfer(val);  // 数值
    CS_HIGH();
}

uint8_t read_reg(uint8_t addr) {
    uint8_t result;
    CS_LOW();
    spi_transfer(addr | 0x80); 
    result = spi_transfer(0xFF); // 此时获取结果
    CS_HIGH();
    return result;
}
```

## 6. 避雷指南

1. **CS 线管理**：确保命令、地址在同一个 CS 周期。
2. **频率限制**：从低频开始调试。
3. **高阻态**：释放未选中从机的 MISO。

## 1. 协议本质

**SPI (Serial Peripheral Interface)** 是一种**同步、全双工、主从式**的串行通信协议。

*   **同步**：通信双方共用一个时钟信号（SCLK），控制收发节拍。
*   **全双工**：主机发送数据时，从机可同时回发数据，总线是双向的。
*   **主从结构**：主机（Master）带动从机（Slave），时钟永远由主机产生。

---

## 2. 硬件接口信号线

| 信号线 | 全称 | 功能描述 | 方向 |
| :--- | :--- | :--- | :--- |
| **SCLK** | Serial Clock | **时钟脉冲**。控制数据位传输的速度和时机。 | 主 → 从 |
| **MOSI** | Master Out Slave In | **主发从收线**。主机发送给从机的信号。 | 主 → 从 |
| **MISO** | Master In Slave Out | **主收从发线**。从机返回给主机的信号。 | 从 → 主 |
| **CS/SS** | Chip Select | **片选信号**。低电平有效，用于选择目标从机。 | 主 → 从 |

## 3. 核心时序：模式分类

SPI 由 **CPOL (极性)** 和 **CPHA (相位)** 组合成 4 种模式：

| 模式 | CPOL | CPHA | 空闲电平 | 采样边沿 |
| :--- | :--- | :--- | :--- | :--- |
| **Mode 0** | 0 | 0 | 低 | 第 1 边沿 (上升) |
| **Mode 1** | 0 | 1 | 低 | 第 2 边沿 (下降) |
| **Mode 2** | 1 | 0 | 高 | 第 1 边沿 (下降) |
| **Mode 3** | 1 | 1 | 高 | 第 2 边沿 (上升) |

---

## 4. 数据交互：位交换机制

SPI 的物理本质是两个**移位寄存器**的互换。
*   **每一个 SCLK 脉冲**，主机移出 1 位到 MOSI，同时从机移出 1 位到 MISO。
*   **8 个脉冲后**，主从机完成 1 字节的互换。
*   **推论**：读数据必须通过发送“哑数据”（如 0xFF）来开启时钟。

## 5. 软件驱动开发思路

### 5.1 字节级互换
```c
uint8_t SPI_Transfer(uint8_t tx_data) {
    SPI_DR = tx_data;              // 写入发送寄存器
    while(!TXE_FLAG);              // 等待发送完成
    while(!RXNE_FLAG);             // 等待接收完成
    return SPI_DR;                 // 返回接收到的字节
}
```

### 5.2 寄存器操作
- **写**：CS=0 -> 发地址 -> 发数据 -> CS=1。
- **读**：CS=0 -> 发地址 -> 发 Dummy -> 存结果 -> CS=1。

## 6. 工程避雷指南

1.  **CS 边界**：一次完整的事务（命令+数据）必须保持 CS 持续低电平。
2.  **MISO 冲突**：一主多从时，未选中从机必须释放 MISO 为高阻态。
3.  **频率限制**：配置 SCLK 频率不可超过从机手册标称的最大值。
4.  **位序模式**：确认软件配置与硬件要求的 **MSB First** 是否一致。
